General Workflow

Create a Flare project locally

Create an empty remote repository through a provider (GitHub, BitBucket, GitLab, CloudForgeGit)

Get the remote repository URL (should end in .git)

Bind through Flare using Project Ribbon, Project Properties, and Source Control Tab

*Demo in Flare and GitHub*

Committing Changes/Pushing Changes

Make some edits and save files.  You should see a red check mark.  This means there are changes that are not committed

Commit changes – commits it to local repository – check mark goes away – remote repository does not have it yet because it is not pushed

Push to remote repository (or Synchronize if working with multiple users)

Importing from Source Control

A project is already uploaded on a remote repository

Browse to the remote repository (typically through the browser, through www.github.com or www.bitbucket.com)

Get the URL (should end in .git)

Start a New Project > Import from Source Control >  Git > Paste URL > Select Project Files

Commiting/Pushing changes outside of Flare (when Flare isn’t working)

  1. Open Git Bash in the project root folder
  2. `git status` to see what needs to be committed
  3. `git add .` to add all pending changes to be commited
  4. `git commit - m “more changes”`
  5. `git status` to see no pending changes
  6. `git push` to push to remote server (or `git push origin master`)
  7. Should be able to pull latest repo from repository by hitting ‘pull’, closing the project and reopening and hitting ‘yes’, or reimport from source control

Using Git with SSH Keys

  1. Open Git Bash
  2. Type `ssh-keygen`
  3. Follow prompts to store in default location; Should be Users/mtran/.ssh
  4. Navigate to it in Windows (Users/mtran/.ssh)
  5. Open id_rsa.pub in notepad and copy the entire content
  6. In your Git provider, go to Settings and add SSH key
  7. Pull git repo through import through source control, and choose SSH key files by browsing

Publishing to Git

  1. Do not publish to the same repository that project is bound to
  2. Set up a separate ‘Output/Publish’ Repository
  3. Create New Destination
  4. Choose Source Control from the dropdown
  5. Create a temporary folder on machine outside of the Flare project (like desktop or documents)
  1. This basically serves as a local publishing destination, where it will initialize git, then push to the new repository
  1. Insert git url and credentials
  2. Enable push to Remote
  3. Choose Origin
  4. Save
  5. Publish from Target
  6. Check publish destination / remote repository

How to ‘Roll Back/Revert’ to a previous commit

  1. Browse commit history on remote repo and find the ‘commit number’
  2. Close Flare
  3. Open Git bash in root folder
  4. `git reset –hard ########’
  5. Open Flare and see that the project is at the correct state
  6. `git push origin master --force'
  1. This removed the commits after and pushed it over the remote repo, so the remote repo loses the same number of commits

Other issues with Git

  1. Someone had a .git folder in a parent folder
  1. So the entire folder (like My Projects) had a .git file and had git initialized
  2. Possibly a `git init` run from that folder
  3. Check to see if that parent folder was intended to be initialized in git
  4. Easy to spot because when you do `git status` you see a lot of the other folders outside of the Flare project
